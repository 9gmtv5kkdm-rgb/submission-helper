# 投稿前检查清单 (Pre-Submission Checklist)

> 7大类45项，逐条核对，避免因细节问题被 desk rejection。所有示例中的期刊名、论文信息均为**虚构**。

---

## 使用说明

- 每条检查项前有 `[ ]` 勾选框，完成后填入 `[✓]`
- "常见坑" 列说明了每项最容易出问题的地方
- 中文学报特有要求在最后单独列出

---

## 一、格式合规 (Formatting Compliance) — 10项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 页边距符合期刊要求（常见：1 inch / 2.54 cm 或 A4 特定边距） | 从 LaTeX 默认模板直接导出，未检查期刊是否指定了不同的页边距 |
| [ ] | 字体和字号符合期刊要求（如 Times New Roman 12pt） | 混用多种字体；图表中用了正文不支持的字体 |
| [ ] | 行距符合要求（常见：双倍行距 double-spaced） | 参考文献页忘了改行距；表格内行距与正文不一致 |
| [ ] | 所有页面连续编号 | 封面页未编号或使用了罗马数字；补充材料未编号 |
| [ ] | 标题页格式符合要求（标题/作者/单位/通讯信息/页眉Running title） | 通讯作者邮箱写错；作者单位顺序与署名顺序不一致 |
| [ ] | 正文总字数/页数在期刊限制内 | 引言或讨论部分超长；字数统计方式（含不含参考文献/图表标题）与期刊要求不一致 |
| [ ] | 摘要字数在期刊限制内（常见：150-300 words） | 结构化摘要与非结构化摘要搞混；关键词数量超出限制 |
| [ ] | 参考文献格式符合期刊要求（APA/Vancouver/Chicago/GB-T 7714 等） | 混用多种引用格式；期刊缩写与全称不统一 |
| [ ] | 章节标题层级和编号符合期刊模板 | 引言编号为"0"或编号从"2"开始（部分期刊要求引言不编号） |
| [ ] | 所有缩写首次出现时有全称定义 | 摘要中使用的缩写未定义（多数期刊要求摘要中也要定义，即使正文会再次定义） |

## 二、内容完整 (Content Completeness) — 8项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 标题页完整：论文标题、所有作者全名、单位、ORCID（如要求）、通讯作者信息 | 作者署名遗漏中间名缩写；单位地址不完整 |
| [ ] | 摘要页：摘要正文 + 关键词（3-8个） | 关键词太泛（如 "machine learning"），缺乏区分度 |
| [ ] | 正文结构完整：Introduction / Methods / Results / Discussion / Conclusion | Discussion 和 Conclusion 内容大量重复 |
| [ ] | 致谢（Acknowledgments）部分的基金编号准确无误 | 基金编号少一位；多个基金未全部列出 |
| [ ] | 作者贡献声明（Author Contributions，CRediT 格式） | 某位作者没有对应的贡献类型；贡献描述过于笼统 |
| [ ] | 利益冲突声明（Declaration of Competing Interests） | 遗漏间接利益冲突（如合作者担任该刊编委） |
| [ ] | 基金信息（Funding）完整，含基金名称和 grant number | 基金名称与基金委官网公布名称不一致 |
| [ ] | 数据可获取声明（Data Availability Statement） | "Data available upon request" 不够具体；实际数据不可获取但未说明原因 |

## 三、图表规范 (Figures & Tables) — 7项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 所有图片分辨率 ≥ 300 dpi（打印期刊通常要求 600-1200 dpi） | 截图类图片分辨率不足；从 PPT 导出图片分辨率只有 96 dpi |
| [ ] | 图片格式符合要求（TIFF/EPS/PDF 为常见可接受格式） | 提交了 JPEG 格式导致压缩伪影；矢量图错误导出为位图 |
| [ ] | 图表编号连续，无遗漏或跳跃 | 删除某张图后未更新后续编号；正文引用使用了旧编号 |
| [ ] | 正文中对所有图表均有引用，引用顺序与图表编号顺序一致 | 图4在文中先于图3被引用 |
| [ ] | 图题（Figure Caption）完整且独立于正文可理解 | 图题中的缩写未解释；缺少必要的统计信息（n值、误差线含义） |
| [ ] | 表题在表的上方，图题在图的下方（遵循期刊惯例） | 中英文期刊排版要求不同，搞反了图题和表题的位置 |
| [ ] | 彩色图片在黑白打印下仍可区分（如适用） | 仅用颜色区分不同数据系列，无填充纹理或线型区分 |

## 四、参考文献 (References) — 6项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 所有正文引用在参考文献列表中均有对应条目，反之亦然 | 修改文章时删了某条引文但忘了删参考文献条目；或加了条目忘加引用 |
| [ ] | 引用格式统一（作者-年份编号与顺序编号不混用） | 从不同来源复制参考文献，格式未统一 |
| [ ] | DOI 完整且可访问（特别是 in-press 或 early access 文献） | 使用预印本的 DOI 而非正式版的 DOI；DOI 链接失效 |
| [ ] | 期刊名缩写/全称一致（按期刊要求） | 同一期刊在参考文献中用了两种不同的缩写形式 |
| [ ] | 检查"零引用"：是否有正文主张需要但未引用文献支撑 | 方法部分的参数选择无引用；"研究表明" 后没有引用 |
| [ ] | 自引比例合理（一般不超过总引用的 20-30%） | 无合理原因的自引率过高会引起编辑警觉 |

## 五、伦理合规 (Ethical Compliance) — 5项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 涉及人类参与者/动物的研究已注明伦理审批编号和机构 | 单位内部项目未走伦理审批但论文要求声明；审批编号写错 |
| [ ] | 涉及人类参与者的研究已说明知情同意获取方式 | 仅写"informed consent was obtained"但未说明是书面还是口头 |
| [ ] | 数据可获取声明真实可兑现 | 声称数据公开但实际未上传或链接失效 |
| [ ] | 代码开源声明（如适用）与实际情况一致 | 声称开源但仓库为空或 README 不完整 |
| [ ] | 版权协议/图片使用授权（如使用已发表图片）已获取 | 引用他人图表未获取版权许可（部分期刊要求上传授权书） |

---

## 六、投稿材料 (Submission Materials) — 6项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | Cover Letter — 按要求撰写并上传（或粘贴到投稿系统文本框） | 用其他期刊的 Cover Letter 模板，期刊名忘了改 |
| [ ] | Title Page — 独立文件或与正文合并（按期刊要求） | 双盲审稿的期刊要求 Title Page 单独上传但合并在正文中 |
| [ ] | Manuscript (Main Document) — 含标题/摘要/正文/参考文献（按期刊要求确定是否含图表） | 图表未嵌入正文也未单独上传，导致审稿人看不到图表 |
| [ ] | Figures — 每张图独立文件上传（按期刊要求） | 文件命名混乱（如 "Figure1_final_v3_revised.png"） |
| [ ] | Tables — 每个表独立文件或嵌入正文（按期刊要求） | 表以图片格式提交而非可编辑格式（部分期刊要求可编辑表格） |
| [ ] | Supplementary Materials — 补充文件按要求整理 | 补充材料无索引/目录；文件太大超出期刊限制 |

---

## 七、期刊特定要求 (Journal-Specific Requirements) — 3项

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 推荐审稿人（如期刊要求）：已按要求提供 2-5 名推荐审稿人及其联系方式 | 推荐的审稿人存在利益冲突（合作者/同单位）；推荐邮箱为个人邮箱而非机构邮箱 |
| [ ] | 回避审稿人（如需）：已列出需要回避的审稿人及理由 | 回避理由不充分（如仅写"个人原因"而无学术性理由） |
| [ ] | 数据共享政策合规：数据已按要求存入指定数据库并获取 accession number | 存入通用仓库（如 Figshare）而非期刊指定的领域数据库（如 GenBank） |

## 附：中文学报特有要求

如果你投稿的是**中文学报**（国内中文核心期刊），额外检查以下项目：

| # | 检查项 | 常见坑 |
|---|--------|--------|
| [ ] | 单位介绍信/投稿证明 —— 加盖单位公章 | 通讯作者和第一作者不在同一单位，不知道用哪个单位的章 |
| [ ] | 版权转让协议 —— 全体作者签名（部分期刊投稿时即要求签署） | 某位作者漏签；用电子签但期刊要求手写签 |
| [ ] | 作者贡献声明 —— 中文版本，注明每位作者的具体贡献 | 仅提供了英文 CRediT 声明，未提供中文版本 |
| [ ] | 基金证明 —— 提交基金批文复印件或基金编号 | 基金已结题但论文标注为在研 |
| [ ] | 中文摘要和英文摘要均符合字数要求 | 中文摘要翻译味过重；中英文摘要的关键词不完全对应 |
| [ ] | 中图分类号 / 文献标识码（如期刊要求） | 分类号选错学科大类 |
| [ ] | 图表标题中英文双语 | 图题用了英文但表题用了中文，不一致 |
| [ ] | 参考文献格式为 GB/T 7714（或期刊指定的中文引用格式） | 英文文献和中文文献的 GB/T 7714 格式要求不同，混用 |

---

## 检查清单使用建议

1. **投稿前 7 天**完成第一轮全项检查
2. **投稿前 1 天**完成第二轮复查（重点关注第一轮漏掉的项目）
3. **投稿前 1 小时**快速扫一遍#一（格式合规）和#六（投稿材料），避免低级错误
4. 检查清单另存为 checklists/your-paper-name.md，作为此次投稿的过程记录
